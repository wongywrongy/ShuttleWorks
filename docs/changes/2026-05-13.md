## Step 1 — Persistence layer

### Summary
Replaced the JSON-file persistence (`services/persistence.py` + `services/_backups.py`) with SQLAlchemy 2.0 sync + Alembic + SQLite for the scheduler product. Routes now go through a repository layer (`backend/repositories/`) instead of a `PersistenceService` global.

### Files added
- `backend/app/config.py` — Pydantic `Settings` (database_url + Supabase placeholders + cors_origins).
- `backend/database/{__init__,models,session}.py` — SQLAlchemy `DeclarativeBase` and three tables: `tournaments`, `match_states`, `tournament_backups`. Engine + sessionmaker bound to `settings.database_url`; SQLite path uses `StaticPool` for in-memory tests.
- `backend/repositories/{__init__,base,local}.py` — `Protocol` interfaces per entity + a sync `LocalRepository` façade with three sub-repos plus three orchestration methods (`commit_tournament_state`, `snapshot_current_tournament`, `restore_tournament_from_backup`) that preserve the legacy "backup-before-write" + rotation-to-10 semantics.
- `backend/alembic/`, `backend/alembic.ini`, `backend/alembic/versions/c6361600d776_step1_initial_persistence.py` — initial migration covering all three tables.
- `tests/_helpers.py` — `isolate_test_database(tmp_path, monkeypatch)` + `purge_backend_modules()`. Tests import these directly.
- `tests/unit/__init__.py`, `tests/unit/test_repositories.py` — 23 unit tests against `sqlite:///:memory:` covering CRUD on each entity and all orchestration methods (incl. FK cascade-delete).

### Files modified
- `backend/requirements.txt` — added `sqlalchemy>=2.0`, `alembic>=1.13.0`, `pydantic-settings>=2.1.0`.
- `backend/app/main.py` — lifespan now runs `alembic upgrade head` on startup (idempotent); removed `app.state.persistence = PersistenceService()`.
- `backend/api/tournament_state.py` — rewritten thin handlers calling `repo.commit_tournament_state` / `repo.snapshot_current_tournament` / `repo.restore_tournament_from_backup`. Sync `def` routes.
- `backend/api/match_state.py` — rewritten with DTO↔ORM translation helpers (`_dto_to_fields`, `_row_to_dto`). `_require_tournament_id` auto-creates an empty singleton tournament on first match-state write (legacy parity; retires in Step 2 when route URLs gain explicit `tournament_id`).
- `backend/api/schedule_proposals.py`, `schedule_advisories.py`, `schedule_director.py`, `schedule_suggestions.py` — `Depends(get_persistence)` → `Depends(get_repository)`; `await svc.read_X()` → sync `repo.X.get_Y()`. ### Files removed
- `backend/services/persistence.py` — replaced by repositories.
- `backend/services/_backups.py` — file-rotation helper, no longer needed.
- `tests/test_schedule_state_migration.py` — tested v1→v2 of the on-disk JSON shape; the v1 path no longer exists.

### Tests removed (JSON-file implementation details with no SQL analogue)
Dropped from `test_tournament_state.py` and `test_match_state.py` (originally ~15 tests):
- SHA-256 `_integrity` field stamping / tampering / truncation recovery
- Corrupt-file auto-recovery from disk backup
- Newest-backup-corrupt cascading recovery
- File-based backup rotation count on disk
- Legacy-file-without-integrity load
- Key-order-independent integrity hash

The behaviours they tested (file-level integrity and recovery) don't exist on the SQL path; SQLite has its own page-level integrity and we no longer write rolling JSON files. Backup *semantics* (list / create / restore / rotate to 10) are still tested at the API level and in `tests/unit/test_repositories.py`.

### Decisions made during execution
- **Backup model preserved as a SQL table** (`tournament_backups`) rather than dropped. Kept the legacy API contract and frontend BackupPanel intact; rotation enforced in `LocalRepository.commit_tournament_state` (keep last 10).
- **Routes stay `async def`** but call sync repository methods. The tech-stack-doc "sync def routes via threadpool" end-state is achievable later without semantic change. Step 1's scope is the persistence swap.
- **`match_state.py:_require_tournament_id` auto-creates** an empty singleton tournament when none exists. Preserves the legacy "PUT /match-states without prior /tournament/state PUT" affordance. Retires in Step 2.
- **`open_repository()` context manager** added so background workers (suggestions worker) can get a sync session/repo outside of FastAPI request scope.
- **Python 3.13.7** runtime locally (`python_requires` is 3.11 per tech-stack); all deps install cleanly. Docker still pins 3.11 per `backend/Dockerfile`.

## Step 2 — Multi-tournament API + frontend migration

### Summary
End of the singleton-tournament era. Every state-touching backend route now lives under `/tournaments/{tournament_id}/...`; the new `/tournaments` CRUD endpoints power a list page at `/` and a per-tournament wrapper at `/tournaments/:id/*`. In-process proposal + suggestion stores are nested by `tournament_id`. The `SuggestionsWorker`'s `TriggerEvent` carries the id explicitly; the old 90 s heartbeat tick retired.

### Files added
- `backend/api/tournaments.py` — `GET/POST /tournaments`, `GET/PATCH/DELETE /tournaments/{id}`, plus scoped state + backup endpoints (`/state`, `/state/backups`, `/state/backup`, `/state/restore/{filename}`).
- `tests/test_tournaments.py` — 24 endpoint tests covering CRUD, scoped state, backup rotation, cross-tournament isolation.
- `frontend/src/hooks/useTournamentId.ts` — `useTournamentId()` / `useTournamentIdOrNull()` URL-param readers. The whole frontend reads `id` from React Router; no module-level "active tournament" in the api client.
- `frontend/src/pages/TournamentListPage.tsx` — minimal dashboard list + "New Tournament" modal.
- `frontend/src/pages/TournamentPage.tsx` — `/tournaments/:id/*` wrapper; sets `useUiStore.activeTournamentId` for module-level helpers (`forceSaveNow`).

### Files modified
- `backend/repositories/{base,local}.py` — replaced `get_singleton` / `upsert_singleton` with `list_all`, `create`, `get_by_id`, `update`, `delete`, `upsert_data`. Orchestration methods (`commit_tournament_state`, `snapshot_tournament`, `restore_tournament_from_backup`) take an explicit `tournament_id`. First-write-skip-backup is now keyed on `data == {}` (newly created rows).
- `backend/api/match_state.py` — router prefixed `/tournaments/{tournament_id}/match-states`. `_require_tournament_id` retired; 404 on missing tournament.
- `backend/api/schedule_proposals.py` — router prefixed `/tournaments/{tournament_id}/schedule/proposals`. `_get_store(app, tid)` and `_get_suggestion_store(app, tid)` now nest by tournament; `_read_persisted_state(repo, tid)` and `_persist_committed_state(repo, tid, ...)` take the id. The lock stays app-wide (single uvicorn worker, sub-ms contention).
- `backend/api/schedule_advisories.py` — scoped under `/tournaments/{tournament_id}/schedule`. REPAIR-trigger fingerprints now include `tournament_id` so different tournaments don't collide.
- `backend/api/schedule_director.py` — scoped + `_apply_*` helpers take `tournament_id`.
- `backend/api/schedule_suggestions.py` — scoped router; worker handlers (`_handle_optimize`, `_handle_repair`) read `event.tournament_id` instead of `get_singleton()`.
- `backend/services/suggestions_worker.py` — `TriggerEvent.tournament_id: Optional[UUID]` field. Production callers always populate it; legacy tests built events without it.
- `backend/app/main.py` — periodic 90 s OPTIMIZE tick removed (didn't generalise to multi-tournament). Registered the new `tournaments` router; dropped the now-deleted `tournament_state` router. `/health/deep`'s schema-version reference moved to `repositories.local.CURRENT_TOURNAMENT_SCHEMA_VERSION`.
- `frontend/src/api/client.ts` — ~20 method signatures gained `tid: string` as their first arg. New CRUD methods: `listTournaments`, `createTournament`, `getTournament`, `updateTournament`, `deleteTournament`.
- `frontend/src/api/dto.ts` — added `TournamentSummaryDTO`, `TournamentCreateDTO`, `TournamentUpdateDTO`, `TournamentStatus`.
- `frontend/src/store/uiStore.ts` — `activeTournamentId: string | null` + `setActiveTournamentId`. Not persisted — re-derived from the URL on refresh.
- `frontend/src/hooks/useTournamentState.ts` — reads `id` from `useParams`; re-hydrates on tournament change; flushes any pending PUT on unmount via `forceSaveNow`; legacy `scheduler-storage` migration removed; resets the Zustand store on a 204 (no state for this tournament) so leftover state from a previously-viewed tournament doesn't leak.

- `frontend/src/app/App.tsx` — router restructured: `/` → `TournamentListPage`, `/tournaments/:id/*` → `TournamentPage` → `AppShell`. `/display` unchanged; legacy `/tracking` and `/live-ops` redirect home.
- `frontend/src/main.tsx` — one-time `scheduler-storage` cleanup block removed (no longer applicable after Step 1).

### Files removed

- `tests/test_tournament_state.py` — coverage subsumed by `test_tournaments.py`.

### Test churn
Every URL in the existing route tests was rewritten under `/tournaments/{tid}/...`. The bulk substitution + per-test `tid = seed_tournament(client)` injection was scripted (mirrors the Step 1 fixture-rewrite playbook). Added `seed_tournament(client)` helper to `tests/_helpers.py`.

`test_schedule_suggestions.py` was partially rewritten to use a module-level sentinel `_TID = uuid.uuid4()` for unit tests (no DB), keeping the in-memory store-nesting tests fast.

### Decisions made during execution
- **Explicit `tid` first arg over module-level "active tournament" in api client.** The advisor flagged the latter as a regression: `TournamentListPage` has no active tournament; polling hooks race on navigation; Step 6 needs per-row data without "activating" anything. Stuck with the plan.
- **`useTournamentId()` helper** centralises the `useParams<{ id }>()` read. Throws by default; `useTournamentIdOrNull()` for hooks that can run on the public display.
- **`forceSaveNow()` reads the active tournament from `useUiStore`.** The hook (`useTournamentState`) sets it on mount; module-level export keeps a thin API for "save before navigation" without threading `tid` through every caller.
- **`/display?id=<tid>`** — the public display was previously a singleton consumer; with multiple tournaments it needs an explicit id. Picked a query param over a path segment to avoid a second router level; Step 6 will likely add tournament-id awareness here properly.
- **`SuggestionsWorker.TriggerEvent.tournament_id: Optional[UUID]`** kept optional for backward compat with tests that build events directly; production callers (commits, advisories) always populate it.
- **Periodic OPTIMIZE tick dropped from `app/main.py`'s lifespan.** The 90 s heartbeat was single-tournament implicit; commits and advisories now post explicit per-tournament events, so the heartbeat lost its meaning.
