## Step 1 тАФ Persistence layer

### Summary
Replaced the JSON-file persistence (`services/persistence.py` + `services/_backups.py`) with SQLAlchemy 2.0 sync + Alembic + SQLite for the scheduler product. Routes now go through a repository layer (`backend/repositories/`) instead of a `PersistenceService` global.

### Files added
- `backend/app/config.py` тАФ Pydantic `Settings` (database_url + Supabase placeholders + cors_origins).
- `backend/database/{__init__,models,session}.py` тАФ SQLAlchemy `DeclarativeBase` and three tables: `tournaments`, `match_states`, `tournament_backups`. Engine + sessionmaker bound to `settings.database_url`; SQLite path uses `StaticPool` for in-memory tests.
- `backend/repositories/{__init__,base,local}.py` тАФ `Protocol` interfaces per entity + a sync `LocalRepository` fa├зade with three sub-repos plus three orchestration methods (`commit_tournament_state`, `snapshot_current_tournament`, `restore_tournament_from_backup`) that preserve the legacy "backup-before-write" + rotation-to-10 semantics.
- `backend/alembic/`, `backend/alembic.ini`, `backend/alembic/versions/c6361600d776_step1_initial_persistence.py` тАФ initial migration covering all three tables.
- `tests/_helpers.py` тАФ `isolate_test_database(tmp_path, monkeypatch)` + `purge_backend_modules()`. Tests import these directly.
- `tests/unit/__init__.py`, `tests/unit/test_repositories.py` тАФ 23 unit tests against `sqlite:///:memory:` covering CRUD on each entity and all orchestration methods (incl. FK cascade-delete).

### Files modified

### Files removed

- `backend/services/_backups.py` тАФ file-rotation helper, no longer needed.

### Tests removed (JSON-file implementation details with no SQL analogue)
Dropped from `test_tournament_state.py` and `test_match_state.py` (originally ~15 tests):
- SHA-256 `_integrity` field stamping / tampering / truncation recovery
- Corrupt-file auto-recovery from disk backup
- Newest-backup-corrupt cascading recovery
- File-based backup rotation count on disk
- Legacy-file-without-integrity load
- Key-order-independent integrity hash

The behaviours they tested (file-level integrity and recovery) don't exist on the SQL path; SQLite has its own page-level integrity and we no longer write rolling JSON files. Backup *semantics* (list / create / restore / rotate to 10) are still tested at the API level and in `tests/unit/test_repositories.py`.

### Decisions made during execution
- **Backup model preserved as a SQL table** (`tournament_backups`) rather than dropped. Kept the legacy API contract and frontend BackupPanel intact; rotation enforced in `LocalRepository.commit_tournament_state` (keep last 10).
- **Routes stay `async def`** but call sync repository methods. The tech-stack-doc "sync def routes via threadpool" end-state is achievable later without semantic change. Step 1's scope is the persistence swap.
- **`match_state.py:_require_tournament_id` auto-creates** an empty singleton tournament when none exists. Preserves the legacy "PUT /match-states without prior /tournament/state PUT" affordance. Retires in Step 2.
- **`open_repository()` context manager** added so background workers (suggestions worker) can get a sync session/repo outside of FastAPI request scope.
- **Python 3.13.7** runtime locally (`python_requires` is 3.11 per tech-stack); all deps install cleanly. Docker still pins 3.11 per `backend/Dockerfile`.
