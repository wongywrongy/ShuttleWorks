## Step 4 — Supabase Auth

### Summary
Authentication scaffolding wired end-to-end. The backend grows a `get_current_user` dependency that verifies the JWT against Supabase and is attached to every data router; the frontend gets a Supabase client, an `AuthProvider`, a `LoginPage`, an `AuthGuard`, and an axios interceptor that attaches the JWT to every request. A `SUPABASE_URL=""` / `VITE_SUPABASE_URL=""` toggle keeps local dev + pytest working without a real Supabase project.

### Files added
- `backend/app/dependencies.py` — `get_current_user(credentials=Depends(HTTPBearer(auto_error=False)))`. Lazy-constructs the Supabase client; returns a synthetic local-dev user when `SUPABASE_URL` is blank; maps missing / invalid tokens to HTTP 401.
- `tests/unit/test_dependencies.py` — 7 tests covering both modes: synthetic-user when unconfigured, 401 on missing/invalid token, OK on valid token, missing-email tolerance, `user is None` from the provider.
- `frontend/.env.example` — VITE_API_BASE_URL + VITE_SUPABASE_URL + VITE_SUPABASE_ANON_KEY.
- `frontend/src/lib/supabase.ts` — exports `supabase: SupabaseClient | null` + `isAuthConfigured`.
- `frontend/src/context/AuthContext.tsx` — `AuthProvider` + `useAuth()` hook. Subscribes to `supabase.auth.onAuthStateChange`; synthesises a session/user in local-dev mode.
- `frontend/src/app/AuthGuard.tsx` — redirects to `/login` when no session; no-op in local-dev mode (synthetic session is truthy).
- `frontend/src/pages/LoginPage.tsx` — email/password form + Google SSO button. Shows a banner in local-dev mode.

### Files modified
- `backend/requirements.txt` — added `supabase>=2.0.0`.
- `backend/app/main.py` — every data router now registers with `dependencies=[Depends(get_current_user)]`. `/health` and `/health/deep` stay public; Step 7's `/invites/:token` joins the public set when it lands.
- `frontend/package.json` — added `@supabase/supabase-js` (2.105).
- `frontend/src/api/client.ts` — axios request interceptor attaches `Authorization: Bearer <jwt>` from the current Supabase session. Reads at request time so token refreshes propagate without re-instantiating the client.
- `frontend/src/app/App.tsx` — wraps the router in `<AuthProvider>`; `/login` is public; `/` and `/tournaments/:id/*` are wrapped in `<AuthGuard>`. `/display` stays public so a venue TV doesn't need a login (it already takes the tournament id as a query param).

### Decisions made during execution
- **`SUPABASE_URL=""` → auth disabled** is treated as a first-class mode rather than an error. Why: the spec mandates that the test suite (`Write tests for the dependency (mock the Supabase client)`) keep working, but route tests for the rest of the suite don't have a Supabase project to point at. A hard requirement would force every contributor (and every CI run) to set up a real project just to land an unrelated bug fix. The mode is symmetric on both sides: backend returns a synthetic `local-dev` user, frontend exposes a synthetic session. Step 8 deployment sets the real values and production transparently flips into JWT verification.
- **Auth applied at `app.include_router(dep=...)` rather than per-route.** Keeps the dependency wiring in one file; the individual route handlers don't need to import the dependency. The downside — route-level tests that build minimal apps don't exercise the auth dep — is intentional: those tests focus on route logic, not auth wiring, and the dep is covered exhaustively in `test_dependencies.py`.
- **HTTPBearer `auto_error=False`** so unauthenticated requests reach the dependency body (where local-dev mode can serve them). FastAPI's default behaviour would 403 before our code ran.
- **Axios interceptor reads `supabase.auth.getSession()` at request time** rather than caching the token in a Zustand store. Supabase JS handles refresh internally; reading per-request guarantees we always send the latest `access_token` without an extra subscription.
- **Login page in local-dev mode** renders a clear banner explaining the bypass and a Continue button rather than rendering an inert form — pretending auth works when it doesn't would be confusing during a dev session.

- `pytest products/scheduler/tests/` — **244 passed** (237 from Step 3 + 7 new dependency tests).
- `tsc -p tsconfig.app.json --noEmit` — clean.
- `vite build` — clean; new `index` bundle slightly larger (~187 KB / 59 KB gzip) due to the supabase-js + auth context.

### Known follow-ups (not Step 4)
